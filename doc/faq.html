  1. Porque existe um arquivo COPYING?

    O arquivo COPYING informa a licença de uso do modelo. Esse arquivo é importante pois garante que nenhum dos autores e colaboradores irá processar judicialmente algum aluno que utilizar o modelo. Também é importante porque protege os autores e colaboradores contra qualquer possível dano que este modelo possa causar a algum de seus usuários.

  2. Porque é recomendado remover/modificar o arquivo COPYING?

    A licença no arquivo COPYING é muito liberal (ao estilo BSD, MIT) para poder cobrir os desejos de todos os alunos. Infelizmente se esta licença for mantida ao escrever sua defesa esta é automaticamente aplicada a defesa e por este motivo deve ser alterada (recomenda-se remover o arquivo COPYING pois assim todos os direitos ficam reservados ao autor o que é muito provavelmente o que deseja).

  3. Ao abrir os arquivos .tex visualizo caracteres estranhos como os apresentados abaixo:

    Muito provavelmente você não configurou corretamente o seu editor ou está utilizando um editor que não é suportado. Procure seu editor aqui e veja como configurá-lo.

  4. Ao tentar compilar minha distribuição de LaTeX não encontra o pacote expl3.sty. Como devo proceder?

    O pacote expl3.sty é um primeiro passo em direção ao LaTeX3. Esse pacote foi renomeado para l3kernels.sty e l3packages (http://sourceforge.net/p/miktex/discussion/33790/thread/fbc850e8).

    Pascal Gaggero escreveu um tutorial explicando como resolver esse problema se você estiver utilizando o MikTeX.

  5. Não modifiquei nenhum arquivo e não estou conseguindo gerar o PDF. O que eu faço?

    Envie o arquivo tese.log (como anexo) por e-mail para um dos mantenedores.

    Se o arquivo tese.log não existir, certifique-se que o LaTeX esteja instalado na sua máquina e devidamente configurado.

  6. Estou tendo problemas com imagens e por causa delas não consigo compilar.

    O modelo foi planejado para ser compilado com o pdflatex utilizando o comando:
    $ pdflatex tese.tex

    Se você estiver utilizando outro compilador, tente utilizar o pdflatex.

  7. Tenho figuras .eps e por causa delas não consigo compilar.

    O modelo foi planejado para não suportar figuras no formato .eps pelo simples motivo dele ter sido "substituído" pelo formato .pdf como verificado abaixo:

    The moral to this story is that PDF can be used as a replacement file format for EPS, and that PDF can be used as a delivery format for sending complete publications to press. (Retirado de http://www.adobe.com/print/features/psvspdf/.)

    Se você tiver gerado as figuras a partir de algum software, muito provavelmente, conseguirá gerá-las no formato .pdf. No caso de você não conseguir gerá-las no novo formato, utilize o epstopdf para criar um .pdf a partir do .eps.

  8. Ao compilar as referências bibliográficas não aparecem.

    Você muito provavelmente está esquecendo de utilizar o bibtex ou equivalentes, e.g., bibtex8 e biber.

    Em resumo, o processo de compilação do modelo consiste nos comandos:
    $ pdflatex tese.tex
    $ bibtex tese.aux
    $ makeindex -s tese.ist tese.idx
    $ pdflatex tese.tex
    $ pdflatex tese.tex

    Recomenda-se utilizar o Makefile presente no modelo ou compilar utilizando o latexmk com o seguinte comando:
    $ latexmk -pdf tese

    O latexmk roda em todas as plataformas e é incluído na maioria das distribuições de LaTeX. Infelizmente, a grande maioria dos editores não estão configurados para utilizá-lo.

  9. Pelas normas da ABNT os apêndices e anexos devem aparecer depois das referências bibliográficas mas no modelo isso não é seguido.

    As dissertações e teses defendidas na UNICAMP não são obrigadas a seguir a risca as normas da ABNT e em várias convenções, e.g, Chicago, Hart e Butcher, as referências bibliográficas aparecem depois dos apêndices e anexos. Além disso, os comando \appendix e \annex que precede os apêndices e anexos, respectivamente deve preceder o comando \backmatter, que por sua vez precede a referência bibliográfica.

  10. Ao utilizar o pacote listings para incluir um código os sinais de menos, -, não aparecem.

    Esse é um bug conhecido ao utilizar os pacotes breqn e listings em conjunto. Não existe previsão dele ser resolvido mas algumas soluções paliativas são descritas em http://tex.stackexchange.com/questions/6827/minus-signs-vanish-with-listings-and-breqn-any-solutions.

  11. Ao tentar compilar minha distribuição de LaTeX não encontra o pacote expl3.sty. Como devo proceder?

    O pacote expl3.sty é um primeiro passo em direção ao LaTeX3. Esse pacote foi renomeado para l3kernels.sty e l3packages (http://sourceforge.net/p/miktex/discussion/33790/thread/fbc850e8).

    Pascal Gaggero escreveu um tutorial explicando como resolver esse problema se você estiver utilizando o MikTeX.

  12. Não consigo compilar devido ao erro Package biblatex Error: File 'tese.bbl' not created by biblatex.. Como resolvo esse problema?

    Talvez uma versão anterior desse arquivo já exista e esteja causando problemas. Procure pelo arquivo tese.bbl e remova-o.

  13. Não consigo compilar a bibliografia porque o biber não funciona. Ele informa data source not found in. Como faço ele funcionar?

    Como Andrew Goldstone reportou em seu blog o biber parece possuir um bug que algumas vezes ele perde o arquivo temporário.

    Considerando que o erro abaixo tenha sido informado:
    data source /tmp/par-72616e69657265/cache-371b0cf3aa3eec4dfb0c82647515336d327c4e62//inc/lib/Biber/LaTeX/recode_data.xml not found in .
    para fazer o biber voltar a funcionar você deve remover os arquivos presentes na pasta /tmp.
